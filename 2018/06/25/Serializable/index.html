JAVA序列化

🍊AwfulOrange
2018-06-25
JAVA
  • 本文主要探讨java默认的序列化方式

参考文章

https://www.geeksforgeeks.org/serialization-in-java/

概念

序列化：将JAVA对象转换成二进制流
反序列化：将二进制流转换为内存中的JAVA对象
这种机制用于持久化对象

Serialization is a mechanism of converting the state of an object into a byte stream. Deserialization is the reverse process where the byte stream is used to recreate the actual Java object in memory. This mechanism is used to persist the object.

如何序列化一个对象？

1、实现java.io.Serializable 接口
2、ObjectOutputStream类的writeObject(Object obj)方法对对象进行序列化

3、ObjectInputStream类的readObject(Object obj)方法对对象进行反序列化

序列化的优点

1、保存/持久化 对象的状态
2、在网络中传递序列化后的对象
只有实现了序列化接口的对象可以被序列化。
Serializable是一个 marker interface（标识接口）- 即接口中没有数据及方法。它用来标记JAVA类 从而使这些类拥有特定的功能
其他的标记接口还有:Cloneable and Remote.

一些需要注意的点

1、父类实现了序列化接口，子类不需要再实现序列化接口，相反情况下（子类实现序列化，父类需要再实现序列化接口）则不同。
2、只有非静态数据成员能通过序列化过程保存。
3、静态数据成员（static）和临时数据成员（transient）不会通过序列化过程保存。因此，如果您不想保存非静态数据成员的值，那么将其设置为transient。
4、在反序列化对象时，不会调用对象的构造函数。
5、关联对象必须实现序列化接口。

SerialVersionUID

序列化运行时将版本号与称为SerialVersionUID的每个可序列化类关联，该类在反序列化期间用于验证序列化对象的发送方和接收方是否为该对象加载了与序列化兼容的类。如果接收方为对象加载了一个与相应发送方类的UID不同的类，反序列化将报出InvalidClassException异常。可序列化的类可以通过声明字段名显式地声明自己的UID。
它必须是静态的、最终的和长类型的。（static final long）

若可序列化的类没有声明serialVersionUID,那么序列化运行时将根据Java对象序列化规范所述，按照类的各个方面计算该类的默认serialVersionUID。
然而，强烈建议所有可序列化的类显式声明serialVersionUID值，因为它的计算高度敏感于类的细节，这些细节可能因编译器的实现而异，类中的任何更改或使用不同的id可能会影响序列化数据。导致异常的抛出。

同时还建议对UID使用private修饰符，因为它不适用于继承的对象。
