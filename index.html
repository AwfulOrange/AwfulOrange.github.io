Postman模拟发送带Cookie的请求

Posted on 2018-04-20

由于chrome的安全限制，postman无法发送带Cookie的请求

如果有需要发送Cookie的需求，需要安装并打开Interceptor拓展程序

安装

  1. 点击图片中Interceptor按钮postman会自动跳转到google商店 在里面的直接点击添加到extension即可

  1. 添加成功后再点击此按钮开启Interceptor即可发送带Cookie的POST请求

方法1

在浏览器中访问你要访问的接口并复制Cookie到Postman

方法2

先访问一遍接口，浏览器获得了Cookie后直接发送Post请求

搭建docker-registry及定时删除image

Posted on 2018-03-29

搭建docker-registry及定时删除image

  • 拉取docker-registry images
  • 添加用户名密码

  • docker stop registry

  • docker-registry 直接开启 （无用户名及密码）

  • docker-registry 添加可删除属性(定时删除过期的docker image 释放磁盘空间)

  • docker-registry http访问

    • ubuntu 14.04(init system: Upstart)

      • sudo vi /etc/docker/daemon.json

      • 添加如下配置后重启（sudo systemctl restart docker）

    • ubuntu 16.04(init system: Systemd)

      • sudo vi /etc/systemd/system/multi-user.target.wants/docker.service
      • ExecStart=/usr/bin/dockerd -H fd:// –registry-mirror=https://6l2qos50.mirror.aliyuncs.com –insecure-registry=192.168.14.190:6000
      • for multi insecure-registry : ExecStart=/usr/bin/dockerd -H fd:// –registry-mirror=https://6l2qos50.mirror.aliyuncs.com –insecure-registry=192.168.14.190:6000 –insecure-registry=192.168.14.190:5000
  • python库删除image
    • https://github.com/andrey-pohilko/registry-cli
  • python ~/registry-cli/registry.py -l bohu:123456 -r http://localhost:5000 –delete
    • 删除image 还需要进行garbage-collect
  • docker exec registry /bin/registry garbage-collect /etc/docker/registry/config.yml
    • 删除物理文件 释放硬盘
  • docker 删除／垃圾回收 dockerimage 问题
    • repush 显示Layer already exists 怀疑cache问题
    • 删除显示：deleting tag 3 tag digest not found: 404
    • docker restart registry 可解决该问题

ubuntu-docker-install

Posted on 2018-03-22

更新apt源

  • sudo apt-get update
  • sudo apt-get install apt-transport-https ca-certificates
  • sudo apt-key adv –keyserver hkp://p80.pool.sks-keyservers.net:80 –recv-keys 58118E89F3A912897C070ADBF76221572C52609D
    • 如果出现无法获取key错误信息，多试几次即可

根据系统选择添加的仓库

ubuntu version Repository
Precise 12.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-precise main
Trusty 14.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-trusty main
Wily 15.10 deb https://apt.dockerproject.org/repo ubuntu-wily main
Xenial 16.04 (LTS) deb https://apt.dockerproject.org/repo ubuntu-xenial main
  • sudo apt-add-repository {Repository}
    • 如果出现：sudo: apt-add-repository: command not found
    • 执行：sudo apt-get install software-properties-common
  • sudo apt-get update

  • sudo apt-cache policy docker-engine（查看所有docker版本）

    • sudo apt-get install -y docker-engine= 按需拷贝对应version
      • sudo apt-get install docker-engine=1.13.1-0~ubuntu-trusty
      • sudo apt-get install docker-engine=1.13.1-0~ubuntu-xenial
      • 升级docker版本
        • 1.12.6 -> 1.13.1
        • sudo apt-get update && sudo apt-get upgrade docker-engine:1.13.1-0~ubuntu-xenial
        • via:https://docs.docker.com/cs-engine/1.12/upgrade/
    • 运行命令后，有很大的几率无法下载，提示速度很慢
    • 可以在/etc/apt/source.list 中将先前倒入的repo的地址中的https改成http 速度会在一定程度上变快

安装完成 添加docker用户

  • sudo usermod -aG docker $USER

  • exit后重新登录后生效

  • 检查docker是否运行

    • Ubuntu 16.04:sudo systemctl status docker
    • Ubuntu 14.04/16.01 : ps aux|grep docker
  • docker image 默认存放目录
    • /var/lib/docker/aufs
